# nanop （nano plus）脚本工具

## 功能

### 编译文件历史记录与重新打开
* **全路径存储**：脚本在保存历史时，会自动将相对路径解析为绝对路径。确保您在系统任何目录下运行 `nanop [编号]` 都能准确打开对应的文件。
* **智能路径显示**：
    * 当文件位于当前目录或其子目录下时，优先显示简洁的相对路径。
    * 能够自动识别并追踪符号链接（Soft Link）的真实物理路径。
* **交互式增强**：
    * 提供 `0)` 选项用于快速新建文件。
    * **异常校验**：对输入的编号进行严格检查，若编号超出历史记录范围或输入非法字符，将提示错误并阻止执行。

### 可选先清空再编辑
* **清空模式**：通过 `-ce` 参数，可以在启动 `nano` 前清空目标文件。此操作会直接作用于文件系统，请谨慎使用。

## 使用

用法: `nanop [选项] [文件名/编号]`

选项:
* `-ce, --clear-edit`  清空文件内容再编辑。
* `-l,  --list`        仅显示历史文件列表（包含最后修改时间与大小）。
* `-c,  --clear-hist`  清空所有已保存的历史记录。
* `-h,  --help`        显示详细的用法说明。

## 交互增强
* 输入 [编号]c        选择该文件并清空编辑
* 输入 [编号]a        选择该文件并跳转到行尾
* 输入 [编号]j[行号]  选择该文件并跳转到指定行号 (示例: 1j50)

## 示例
1. `nanop 1`：快速打开最近编辑过的第一个文件。
2. `nanop -ce /etc/hosts`：以清空模式打开 hosts 文件。
